The Code Factory

An interactive essay by Adam Carruthers

Learn the counterintuitive trick to manage a software team from... manufacturing?

If you're a developer working for some company, you probably have opinions about how your project is being managed. It's natural, we care that our projects succeed. I work for the NHS and it matters a lot that the work we do is successful - what we do can change people's quality of life, it can even save lives. I work on a NHS data system - this system is being used right now to test whether drugs are safe and effective. My work might be making drugs safer.

The success that we want is imperilled when the projects are managed badly. You can throw infinite money at people doing IT but if the work is done the wrong way, you might as well set that money on fire (see: National Programme for IT). Let's fix that by treating IT like a factory, and the work we produce just like products going down a production line.

How is all this relevant? Is this all just an excuse to program a cool graphic in JavaScript? The work of writing code is surely very different to that of screwing on nuts and bolts to a car. I hope to convince you bit by bit that the systematic view that factories use to increase throughput is useful in programming as well.

Let's start thinking like a factory manager. A factory manager knows that:

We create something, ~thing, car?~.
This thing has value, ~money or faster rides to work for customers~.
This value is created by a process.

A factory manager dedicates themselves to understanding how the process creates the value, and then trying to modify the process to make sure it gives the most value. Controversial idea I know.

For us:

We create something, computer programs.
This thing has value, ~money, lives saved, new research, social connections to friends~.
This value is also created by a process.

Let's start process analysing.

Welcome to the factory

Here's a crude simulation of the process of making something. The materials come in from the left, they then go through a series of processing steps or workstations and then leave the building to be shipped to happy customers on the right.

Click the buttons below to add labels to the process to show how it can represent different processes.

Each workstation has a few features. It needs a worker to man it, it needs materials, it uses some machine. Each workstation can process work at a certain speed. Work waiting to be processed sits in a waiting area before the workstation.
